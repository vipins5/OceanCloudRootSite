Before this retirement, when creating an online meeting using the Microsoft Graph API, admins or users can choose whether to require a passcode. If no value is specified, the default behavior does not require a passcode. After this update: For enhanced security, users will still have the option to specify whether a passcode is required when creating a meeting. If no value is specified, the default behavior will require a passcode for the meeting. Effective February 24, 2025, the default value of the isPasscodeRequired property will change from false to true.
