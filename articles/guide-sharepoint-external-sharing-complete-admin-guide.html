SharePoint External Sharing: Complete Admin Guide

Tenant settings, site controls, guest links, Microsoft Entra B2B, and PowerShell checks for safer collaboration.

SharePoint May 25, 2026 4 min read

SharePoint external sharing lets employees collaborate with clients, vendors, and partners. The risk is that the same convenience can expose confidential content if tenant settings, site settings, link defaults, and guest lifecycle controls are not aligned.

How external sharing works

SharePoint uses organization-level and site-level sharing settings. The most restrictive setting always wins: if the tenant blocks anonymous links, a site cannot enable them; if a site is internal-only, users cannot invite guests even when the tenant allows it.

OptionUse caseAdmin guidance
Anyone linksUnauthenticated access by linkAvoid for business data; require expiry if used.
New and existing guestsClient or partner collaborationBest default for controlled collaboration.
Existing guests onlyIT-managed guest accessUse when guests must be pre-approved in Entra ID.
Only people in your organizationInternal sitesUse for HR, finance, legal, executive, and sensitive project sites.

Recommended baseline

Set the tenant to New and existing guests, set the default link to Specific people, and disable external sharing on sensitive sites. For client project sites, use named owners, guest expiry, sensitivity labels, and monthly access reviews.

Admin note: If you disable external sharing and later re-enable it, previous guests can regain access. Remove site-level guest permissions first when you are closing access permanently.

Microsoft Entra B2B integration

Modern SharePoint sharing relies on Microsoft Entra B2B so guests are represented in your directory and can be governed by external collaboration settings. This makes conditional access, domain allow/deny rules, and guest reviews more consistent across Microsoft 365.

PowerShell configuration

Use the SharePoint Online Management Shell for tenant and site checks. Test first, then document changes.

PowerShell 7 — review and configure external sharing
Connect-SPOService -Url "https://contoso-admin.sharepoint.com"

# Review tenant-level sharing posture
Get-SPOTenant | Select-Object SharingCapability, DefaultSharingLinkType, EnableAzureADB2BIntegration

# Set a controlled tenant baseline
Set-SPOTenant -SharingCapability ExternalUserSharingOnly
Set-SPOTenant -DefaultSharingLinkType SpecificPeople
Set-SPOTenant -RequireAnonymousLinksExpireInDays 30

# Enable guest sharing for one collaboration site
Set-SPOSite -Identity "https://contoso.sharepoint.com/sites/client-project" `
  -SharingCapability ExternalUserSharingOnly

Operating controls

  • Assign two accountable site owners for every externally shared site.
  • Use sensitivity labels and DLP policies for regulated content.
  • Run access reviews for guests and remove accounts when the relationship ends.
  • Audit Anyone links and sites where sharing has drifted from policy.
