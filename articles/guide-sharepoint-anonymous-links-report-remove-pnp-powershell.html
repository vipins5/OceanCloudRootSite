Report and Remove Anonymous Sharing Links with PnP PowerShell

Use a safety-first script to audit Anyone links across SharePoint Online and OneDrive, then optionally remove them with explicit confirmation.

Admin & Automation June 12, 2026 12 min read

This guide walks you through running AnonymousLinks_Report_And_Remove.ps1 safely in production. You will start in report-only mode, validate results, and only then enable removal if needed.

Built-in safety: the script defaults to Report mode and requires a second explicit confirmation switch before deleting links.

Download the script

Download the exact script used in this guide:

  • Download AnonymousLinks_Report_And_Remove.ps1

Prerequisites

1) PowerShell version

  • Supported: Windows PowerShell 5.1 and PowerShell 7+ on Windows.
  • Recommended: latest PowerShell 7 for better runtime behavior and module consistency.
  • Download PowerShell: Microsoft official install guide.

2) PnP.PowerShell module

  • Recommended version: latest stable PnP.PowerShell 2.x.
  • Install: Install-Module PnP.PowerShell -Scope CurrentUser
  • Update: Update-Module PnP.PowerShell
  • Check version: Get-Module PnP.PowerShell -ListAvailable | Sort-Object Version -Descending | Select-Object -First 1 Name,Version
  • PnP install docs: pnp.github.io/powershell installation

3) Entra app registration and permissions

You need an Entra application with PnP PowerShell authentication configured.

  • Report-only scans: grant read permissions sufficient for inventory (for example Sites.Read.All).
  • Removal mode: script notes that app-only removal requires broader permission such as Sites.FullControl.All.
  • Grant admin consent before running in production.

If your app is already registered: keep using the same app and skip registration creation. Just verify permissions and proceed.

4) Certificate / auth setup

The script supports:

  • Interactive login (user context)
  • Thumbprint (app-only, certificate in local store)

5) VS Code (optional)

  • Optional editor for reviewing and versioning the script.
  • Download: VS Code official download

Configuration before first run

Open the script and edit the hard-coded variables at the top:

PowerShell - minimum required config values
$TenantName = "contoso"
$ClientId   = "00000000-0000-0000-0000-000000000000"
$AuthMode   = "Thumbprint"   # or "Interactive"
$TenantId              = "contoso.onmicrosoft.com"
$CertificateThumbprint = "XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"

$Mode = "Report"              # Report or Remove
$ConfirmRemoval = $false       # Must be true to allow deletion in Remove mode
$WhatIfRemoval = $false        # Preview only in Remove mode

Recommended rollout: start with Mode = "Report", review CSV output, then switch to Mode = "Remove" only after approval.

How to run the script

  1. Open PowerShell as your admin/automation identity.
  2. Navigate to the folder containing the script.
  3. Run in report mode first.
PowerShell - report-only run (safe default)
Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned
.\AnonymousLinks_Report_And_Remove.ps1

After completion, check the generated files:

  • AnonymousLinks_Report.csv (main findings and actions)
  • AnonymousLinks_Report_Errors.csv (failures by stage)
  • AnonymousLinks_Transcript_yyyyMMdd_HHmmss.txt (execution log)

Removal run (only after validation)

To remove anonymous links, set all of the following in script config:

  • $Mode = "Remove"
  • $ConfirmRemoval = $true
  • $WhatIfRemoval = $false (or $true to preview only)

Scope reminder: according to script notes and PnP behavior, removal cmdlets target file/folder sharing links. List-item link removal is not covered by these commands.

Understanding output actions

ActionMeaning
ReportOnlyLink was detected and logged. No deletion attempted.
WouldRemovePreview mode. Script logged what would be removed.
RemovedAnonymous link deletion succeeded.
RemoveFailedDeletion attempt failed; check error CSV and transcript.
